# Legacy Player — Venture Lab Pitch

## One-line pitch

Legacy Player is a self-hosted multiplayer infrastructure platform for legacy games, enabling classic local-only titles to be played online through emulator adapters, runtime synchronization, and game-specific orchestration.

---

## Problem

A large number of classic multiplayer games were designed only for local play.

These games still have active communities, but modern players face several barriers:

- physical co-location is rare
- existing emulator netplay is fragmented
- setup is difficult
- compatibility is inconsistent
- community hosting infrastructure is weak
- many games require manual workarounds to remain stable online

As a result, strong multiplayer experiences remain locked behind aging hardware assumptions.

---

## Solution

Legacy Player provides a reusable, self-hostable multiplayer runtime for legacy games.

It combines:

- session orchestration
- emulator integration
- game-specific multiplayer logic
- runtime patch assistance
- replay and diagnostics
- self-hosted lobby and relay infrastructure

This allows communities to run classic games online in a way closer to how communities run dedicated servers for modern games.

---

## What makes it different

Legacy Player is not just emulator netplay.

It is:

- game-aware
- self-host infrastructure oriented
- built around reusable runtime components
- designed for replay, diagnostics, and community hosting
- intended to scale through platform adapters and game packs

---

## Initial wedge

Start with:

- GameCube
- Dolphin
- party games
- one flagship title

This gives Legacy Player a strong first proof while staying technically realistic.

---

## Why this can matter

Legacy Player can become:

- a retro multiplayer platform
- a tournament/community infrastructure layer
- a replay archive system
- a preservation tool for multiplayer game history
- a multiplayer research foundation for future engine systems

---

## Users

Potential user groups include:

- retro gaming communities
- tournament organizers
- preservation groups
- emulator power users
- content creators
- developers exploring multiplayer sync systems

---

## Business and product paths

Possible paths include:

- open-core with premium server tooling
- hosted relay/tournament services
- replay and analytics tools
- community infrastructure subscriptions
- SDK/adaptor ecosystem

The core runtime can remain self-hostable while premium infrastructure and tooling can expand separately.

---

## Why now

Legacy gaming communities are large and persistent, while self-host culture is growing again.

Legacy Player fits a current need:

- classic games still matter
- players want community infrastructure
- self-hosting is attractive
- emulator-based ecosystems are mature enough to support adapter-driven tooling

---

## Long-term strategic value

Legacy Player also serves as a multiplayer research and systems foundation for future engine work, especially for CDE.

Instead of waiting to design multiplayer inside a future engine, Legacy Player allows multiplayer architecture to be proven first in a live environment.
